# CONTRIBUTING — Periódico Digital Premium

## Principios

- Mantener calidad editorial y técnica.
- No mezclar responsabilidades de módulos.
- No hardcodear secretos.
- No exponer contenido premium.
- No permitir acciones críticas sin permisos.
- Documentar pendientes en Markdown.

## Convenciones

- Código e identificadores en inglés.
- Documentación en español técnico.
- Commits claros.
- Migraciones para cambios de base de datos.
- Tests o smoke tests cuando aplique.

## Pull requests

Todo PR debe indicar:

- Qué cambia.
- Qué módulos afecta.
- Cómo se probó.
- Qué pendientes quedan.
- Si afecta premium, comentarios, publicidad, pagos o auditoría.
